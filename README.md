# 架電カウンター セットアップ手順

## フォルダ構成（配布時）
```
架電カウンター/
├── 架電カウンター.exe
├── credentials.json   ← サービスアカウントのキーファイル
└── config.json        ← メンバー・列設定
```

## config.json の編集方法

### メンバー追加
`members` 配列に名前を追加するだけ。  
**スプレッドシートのシートタブ名と完全一致させること。**

```json
"members": ["田中", "佐藤", "鈴木", "高橋", "伊藤", "新メンバー"]
```

### 列マッピング変更
実際のスプレッドシートの列が違う場合はここを修正する。

```json
"column_mapping": {
  "総コール":  "B",
  "有効":     "C",
  "再コールA": "E",
  ...
}
```

### data_start_row
データが何行目から始まるか（デフォルト: 3）

## Google Sheets API サービスアカウント 作成手順

### 1. Google Cloud プロジェクト作成
1. https://console.cloud.google.com/ を開く
2. 画面上部の「プロジェクトを選択」→「新しいプロジェクト」
3. プロジェクト名を入力（例: teleapo-counter）→「作成」

### 2. Google Sheets API を有効化
1. 左メニュー「APIとサービス」→「ライブラリ」
2. 検索欄に「Google Sheets API」と入力
3. 「Google Sheets API」をクリック→「有効にする」

### 3. サービスアカウント作成
1. 左メニュー「APIとサービス」→「認証情報」
2. 「認証情報を作成」→「サービスアカウント」
3. サービスアカウント名を入力（例: sheets-writer）→「作成して続行」
4. ロールは「編集者」を選択→「完了」

### 4. JSONキーをダウンロード
1. 作成したサービスアカウントをクリック
2. 「キー」タブ→「鍵を追加」→「新しい鍵を作成」
3. 形式「JSON」を選択→「作成」
4. ダウンロードされた JSON ファイルを `credentials.json` にリネーム
5. exeと同じフォルダに置く

### 5. スプレッドシートを共有
1. スプレッドシートを開く
2. 右上「共有」ボタンをクリック
3. サービスアカウントのメールアドレスを入力
   - 例: `sheets-writer@teleapo-counter.iam.gserviceaccount.com`
   - credentials.json の `client_email` フィールドに記載されている
4. 権限を「編集者」に設定→「送信」

これで準備完了です。

## 使い方
1. exeを起動
2. 名前をドロップダウンから選択
3. 日付を確認（デフォルト: 今日、形式: YYYY-MM-DD）
4. ＋／－ボタンで架電結果を入力
5. ボタンを押した瞬間にスプレッドシートへ自動反映

## 注意事項
- 日付のA列の書式は `5/26(水)` 形式である必要があります
- シートタブ名とconfig.jsonのmembersが一致していること
- ％列（黄色セル）には一切書き込みません
